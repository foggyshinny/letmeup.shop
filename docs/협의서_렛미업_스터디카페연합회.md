## 3. 용어 정의

- **기술 문의**: 키오스크·태블릿·서버·결제 연동·자동문 등 **시스템/장비의 기술적 사항**에 관한 문의
- **사용법 문의**: 관리자 웹·키오스크 화면 **조작 방법, 설정, 운영 노하우**에 관한 문의
- **장애**: 정상 운영이 불가능하거나 회원 입퇴실·결제에 지장을 주는 **긴급 상황**
- **1차 응대**: 문의 접수 후 최초로 고객에게 답변·안내하는 단계
- **에스컬레이션**: 1차 응대에서 해결이 어려워 상위 담당(렛미업 기술팀 등)으로 이관하는 절차

## 4. 역할 분담 (핵심)

| 유형 | 주담당 | 협력 | 대응 시간(목표) |
|------|--------|------|-----------------|
| **기술 문의** | 🛠️ 렛미업 | — | 운영시간 내 ··· 시간 이내 |
| **사용법 문의** | 🤝 렛미업 + 협회 | 상호 | **상시** / ··· 시간 이내 |
| **장애 (긴급)** | 🤝 렛미업 + 협회 | 렛미업 기술팀 에스컬레이션 | **상시** / 접수 즉시 |

**원칙**
- **기술**은 렛미업이 **최종 책임**진다.
- **사용법·장애 1차 대응**은 렛미업과 협회가 **함께 상시로** 맡아, 사장님이 어디로 연락해도 안내를 받을 수 있게 한다.
- 협회는 **현장 경험 기반의 친근한 1차 안내**를, 렛미업은 **정확한 기술 해결**을 담당해 상호 보완한다.

## 5. 고객 응대 프로세스

```
[접수] ──▶ [분류] ──▶ [1차 대응] ──▶ (해결?) ──▶ [종결·기록]
                          │             │
                          │             └─ No ─▶ [에스컬레이션 → 렛미업 기술팀] ─▶ [처리 → 종결·기록]
                          └─ 채널: 채널톡 / 전화 / 협회 창구
```

1. **접수** — 채널톡, 전화(1688-4264), 협회 창구 중 어디로 들어와도 접수
2. **분류** — 기술 / 사용법 / 장애로 구분 (분류 기준 §3)
3. **1차 대응** — 역할 분담(§4)에 따라 담당이 즉시 안내
4. **에스컬레이션** — 1차에서 해결이 어려우면 렛미업 기술팀으로 이관 (기준·연락 방법 협의: ···)
5. **종결·기록** — 처리 결과를 공용 기록(§7)에 남겨 재발·통계에 활용
